PennBook

Code available upon request

You’ve heard of Facebook… now introducing PennBook: a miniaturized Facebook clone made specifically for Penn Students. This project was made for my Scalable Cloud Computing class final project. Users can send friend requests to their classmates, make posts and comments on each other’s walls, and communicate with one another through group chat rooms.



Front End


The front end is designed using HTML with Bootstrap. Javascript with jQuery is used to support dynamic content. Every few seconds, the client makes an AJAX request to the server checking for updated content (new posts, comments, chat messages). If these are found, the webpage is updated accordingly by updating the DOM without having to refresh the page.

Back End


The backend is designed using Javascript that interacts with Amazon's NoSQL database, DynamoDB. Our goal for this project was to make the website scalable, so that if potentially thousands of users signed up, our website would still be functional. To do this we had to design a database schema that allowed for efficient lookups of the data that needed to be displayed.

Database Design


Designing a database with efficient time lookups was not always trivial. In DynamoDB all data is stored in tables which contain items each of which has a partition key which determines how the data is grouped within the table. All items in a group can be efficiently accessed by querying the database with the partition key. However, sometimes content needs to be accessed in two different ways. For example, PennBook needs to access all posts by a specific user, as well as all posts on a specific user’s wall. In this instance it is not clear what the partition key should be to store posts. To solve this problem, we created two different tables, each with a different partition key. This was one of many database design decisions we had to make to ensure our system was scalable.

UNIhub

Code available here

UNIhub is an all in one college application organizer for high school students. It includes easily accessible spaces to store all of the information you would put on your college application, such as test scores, GPA, activities, awards, and even a section to brainstorm essay ideas. Additionally you can search through a list of any college in the United States, and add it to your college list. UNIhub provides information about each of these colleges such as admission rate, average test scores and cost of attendance. This data is gathered from a US Department of Education API. This app is coded in Swift and we used Google’s Firebase as a database.



Cube Solver

Code available here

I created a Java program that finds a minimal solution for a 2x2x2 Rubik’s Cube, of which there are 3,674,160 unique permutations. The program first lets you enter the colors from a real life cube so that it matches the virtual cube. Then after some calculation it displays the optimal solution in the console.



How It Works


Firstly I made a Permutation class that stores a single permutation of the cube. This class stores all of the colors on each sticker of this permutation. The Permutation class also has methods that can modify the colors by simulating a rotation of one of the faces. Each permutation has a unique identifier so that it can be distinguished from other permutations. The program starts by finding all Permutations that are 1 move away from the solved permutations. Then it uses these permutations to find all permutations 2 moves away from the solved permutation. It continues this until there are no more unchecked permutations, in this case 14 moves maximum distance from the solved state. The program organizes each of these permutations by how many moves away it is from being solved. Then it can take the imputed permutation from the user, search for it, and work its way backwards towards a solution. For example if we know that the cube is 12 moves away from being solved, then there must be some move that we can make that is 11 moves away from being solved. In this way we can solve the cube optimally.

Challenges


There are two main problems that I ran into during this project. One was an issue of storage and another was an issue of efficiency. The storage issue had to do with the Permutations class. I was storing permutations in this class as a String representation, which means every time I modified a permutation by making a cube rotation, I was creating a new String object. In terms of storage space this is very costly, so I switched to storing permutations using character representations, which are primitive types. The efficiency issue had to do with how I was checking for repeat permutations. When searching through all possible permutations, it is essential that I know when I am seeing a duplicate, so that I can ignore it. If a permutation first shows up 2 moves away from solved, and then shows up again 3 moves away, I only care that it showed up 2 moves away, because this is the optimal solution. To check for duplicates I used a hashset that stored the unique identifiers of the perumations. This allows for near constant time lookups rather than linear time lookups of permutations. My final algorithm takes around 10 seconds to load all permuations and around 2 seconds to find a particular solution.

Chemistry Adventure

Code available here

Chemistry Adventure is a chemistry themed RPG video game that I created in Java. The game begins with your character in a laboratory. Your character is surrounded by 4 tables each with a different chemistry lab set up on them. The only problem is that each lab is missing one of the necessary materials to complete the lab. Your job is to explore the world around you looking for these materials so that you can complete the virtual labs. Once you have completed these labs you will need to complete some calculations with the data gathered and enter the correct answer to pass the lab. Completion of all 4 labs wins the game.


Labs


In each lab you are presented a set of instructions which requires the user to move around virtual lab equipment to complete the lab. At the end of the lab you will have to complete a calculation.
